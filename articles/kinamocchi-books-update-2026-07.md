当アカウントで公開している YouTube チャンネル自動運営の書籍2冊を、2026年7月時点の最新ワークフローへ全面改訂しました。初版を書いた2026年3〜4月から3ヶ月で、パイプラインの中身は別物と言えるほど変わっています。全40章を現行の構成と突き合わせて監査し、36章 + 書籍設定を更新しています。

## 何が変わったのか（改訂ダイジェスト）

### 1. 最後の手作業だったディープリサーチが自動になった

初版では「動画1本ごとに人間がチャットでディープリサーチを実行する」のが唯一の手作業でした。現在は Claude Code ネイティブのディープリサーチ機能（Opus オーケストレーター）をスケジュール実行から自律起動し、リサーチ → 品質ゲート → PR → マージまで人の合図なしで完結します。数字・日付・人名へのファクトチェックフラグ付与と、一次ソースで裏の取れない主張の自律検証ループが品質を守ります。人間の役割は週1回のレポート確認だけになりました。

### 2. 画像生成は「モデル精度に賭ける」から「検証して直す運用」へ

主エンジンを Gemini 系から gpt-image-2（OpenAI）に移行しました。ただし本質はモデルの乗り換えではなく運用の転換です。どのモデルでも日本語テキストは確率的に崩れるため、「プロンプトで文字列を逐語指定（RENDER VERBATIM）+ 生成後に検証して崩れていたら再生成」という二段構えを標準にしました。Gemini は課金上限時のフォールバックとして残しています。

### 3. AI レビューを外部サービス依存から Claude Code 完結へ

PR ごとに外部 AI レビュアー（GitHub Copilot Code Review / Gemini Code Assist）へ依頼して最大25分待つ運用を撤廃し、Claude Code 内で完結する多層レビュー（機械チェック + CI → 同一セッションのセルフレビュー → 別セッションのフレッシュな文脈での精査 → コンテンツは議論型レビュー）に移行しました。待ち時間ゼロで、Gemini Code Assist のサービス終了（2026年7月）の影響も受けません。

### 4. 週2本公開から平日毎日公開へ

公開ペースは週5本（平日毎日 20:00）に引き上げました。制作は週3本ペースを保ち、在庫バッファで公開を支えます。スケジューラーは毎時起動の24時間ルーティングに再設計し、全スロットが担当タスクの前に15種類の自己修復チェック（PR 放置・速報中断・配信滞留などのリカバリ網）を通します。

### 5. 品質系のアップデート

- 動画の尺を2枠制に（複雑なテーマは15〜20分・単一トピックは8〜12分。水増し禁止）
- 台本の前工程に脚本設計（感情曲線・フック設計）と専門チーム並列レビューを追加
- SNS 配信の再編（Qiita 主役・縦型ショートは Instagram Reels へ自動クロスポスト）

## 改訂した書籍

どちらも既購入の方は追加費用なしで最新版を読めます（Zenn の仕様で改訂が自動反映されます）。

### [Claudeだけで動くYouTubeチャンネル ── 非エンジニアのための完全自動化バイブル](https://zenn.dev/kinamocchi_tech/books/claude-auto-youtube-guide)

非エンジニア向けの一冊（全23章）。GitHub とは何か・Scheduled Tasks とは何か、から始めて、ゼロから同じ仕組みを作るガイドです。今回の改訂で「人間の作業は監視だけ」という現行の姿に前提から書き直しました。第12章「最後の手作業が自動になるまで」が改訂の目玉です。

## おわりに

3ヶ月でここまで変わった要因は、Claude Code 側の進化と「失敗を人が見つける」から「失敗を機械が拾い直す」への設計転換でした。ワークフローは今後も変わり続けるので、書籍も Lean Publishing で追随させていきます。
