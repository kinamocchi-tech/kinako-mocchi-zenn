# Claude Codeとは ── チャットのClaudeとどう違うのか

## Claude（チャット）とClaude Codeの違い

「Claude」という名前のサービスには、大きく分けて2種類あります。

| 比較項目 | Claude（チャット） | Claude Code |
|---------|-----------------|-------------|
| アクセス方法 | claude.ai でブラウザから使う | claude.ai/code または VS Code等から使う |
| 主な用途 | 質問・会話・文章生成・Researchなど | **ファイル操作・コード実行・自動化** |
| ファイル操作 | できない（会話のみ） | **できる**（ファイルの作成・編集・削除） |
| 外部コマンド実行 | できない | **できる**（git, npm など） |
| スケジュール実行 | できない | **できる**（Scheduled Tasks） |
| 適した場面 | 単発の質問・Deep Research | **継続的な作業・自動化** |

本書のシステムは **Claude Code** を使って構築します。

:::message
「チャットのClaude」は普段の質問や会話に、「Claude Code」はファイル操作や自動化に使います。第12章（Phase 2：ディープリサーチ）だけが「チャットのClaude」を使う唯一の工程です。
:::

## Claude Codeが「コンピューターを操作できる」とは

Claude Codeは、あなたのPCや、クラウドのコンピューター上で **実際のプログラムを動かせます**。

例えば：
- ファイルを作成・編集する
- `git push` コマンドを実行してGitHubにデータを送る
- VOICEVOX（音声合成）を起動して音声ファイルを生成する
- Remotion（動画制作）を実行して動画ファイルを書き出す

これがチャットのClaudeとの最大の違いです。チャットのClaudeはあくまで「テキストで返答するだけ」ですが、Claude Codeは「実際にアクションを起こせる」のです。

## Claude Codeのクラウド環境

Claude Codeは **クラウド（インターネット上のコンピューター）** 上で動作します。

このクラウド環境の特徴：
- **あなたのPCが必要ない**: PCを閉じていても動き続ける
- **専用のLinuxコンピューター**: 各セッションごとにクリーンな環境が用意される
- **起動時にセットアップが実行される**: session-start.sh（後述）が環境を自動準備する

:::details session-start.sh とは
セッション（接続）が開始されるたびに自動実行される「環境準備スクリプト」です。
本書のプロジェクトでは、このファイルに以下の設定が書かれています：
- タイムゾーンをJST（日本時間）に設定
- VOICEVOXをインストール・起動（音声合成のため）
- GitHubへの認証情報を設定

これにより、毎回のセッションで「音声合成できる環境」が自動で整います。
:::

:::

## CLAUDE.mdファイルの役割

Claude Codeにとって、**CLAUDE.md** は「このプロジェクトでの行動指針」です。

チャットのClaude に「〇〇してください」と毎回指示するのと同様に、
Claude Code には CLAUDE.md に「このプロジェクトでは〇〇するように」と書いておくことで、
毎回の指示なしに一貫した動作をさせられます。

CLAUDE.md の詳細は第5章で説明します。

次の章では、本書のシステムの核心である **Scheduled Tasks** について学びます。
