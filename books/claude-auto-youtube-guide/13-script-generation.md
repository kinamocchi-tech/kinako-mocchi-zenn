# Phase 3 台本生成 ── キャラクターが語る台本をAIが書く

## このフェーズでやること

Phase 3では、Phase 2のリサーチ結果をもとに **2体のキャラクターが掛け合う台本** を自動生成します。

実行タイミング: **毎日 06:00 JST**（自動実行）

## script-pipelineの働き

`script-pipeline` スキルは以下の処理を自動実行します。

1. **リサーチ確認**: `content/research/V{ID}_research.md` を読み込む
2. **キャラクター設定の確認**: `docs/character-sheet.md` を参照
3. **脚本設計（screenplay-writer）**: 視聴者の感情曲線・フック設計・retention_beatsを `outline.json` として設計
4. **脚本レビュー（screenplay-reviewer）**: 5つの専門チームが並列でレビューしPASS/FAILを判定。FAIL時は自動修正ループ（最大2サイクル）
5. **台本生成**: キャラクターの個性・口調・関係性に基づいて台本を生成
6. **品質チェック**: 尺・ファクトチェック・キャラクター一貫性を自動確認
7. **台本JSONの保存**: `content/scripts/V{ID}_script.json` として保存
8. **GitHub PR作成**: 台本をレビュー用PRとして提出

## 台本のフォーマット

生成される台本は **JSON形式** で保存されます。

`target_duration_min` の値は、テーマの複雑さに応じて選ぶ2つの枠（標準枠15〜20分・コンパクト枠8〜12分）のどちらかに対応します（2026年7月導入の2枠制）。次の例は標準枠を選んだ場合です。

```json
{
  "video_id": "V005",
  "title": "Claude Sonnet 4.6 Adaptive Thinking 完全解説",
  "target_duration_min": 15,
  "sections": [
    {
      "section_id": "opening",
      "lines": [
        {
          "character": "mocchi",
          "text": "きなこ〜！さっきニュース見たんだけど、Claude Codeがまたアップデートされたんだって！",
          "fact_check_flags": []
        },
        {
          "character": "kinako",
          "text": "ええ、Adaptive Thinkingね。私も気になってたのよ。",
          "fact_check_flags": []
        }
      ]
    }
  ],
  "fact_check_flags": [
    {
      "location": "section: features, line: 3",
      "claim": "処理速度が2倍になった",
      "rank": "B",
      "source_url": "https://...",
      "note": "公式ブログに記載あり。ベンチマーク詳細要確認"
    }
  ]
}
```

## 品質ゲート：尺のチェック

台本の品質は複数の観点で自動チェックされます。

最重要チェック項目のひとつが **「尺（時間の長さ）」** です。テーマの複雑さによって、標準枠とコンパクト枠のどちらを使うかが決まります。

| 尺の基準 | 理由 |
|---------|------|
| 標準枠 15〜20分 | 多面的な深掘り・歴史や背景を含む複雑なテーマ向け |
| コンパクト枠 8〜12分 | 単一トピックで完結する速報・比較・使い方向け |
| 最低 8分（不変） | YouTube ミッドロール広告の閾値（8分未満では収益機会が大きく減る） |
| 上限 20分（不変） | 超えると視聴離脱が急増 |

枠はテーマが要求する情報量で選ぶものであり、枠に合わせて内容を水増しすることは禁止されています。尺が不足している場合、Claudeは自動で追加コンテンツを生成します。

## ファクトチェックフラグ

台本内の数値・日付・人名などは **`fact_check_flags`** で明示されます。

フラグのランク:
- **ランクA**: 一次ソース（公式サイト・論文）で確認済み
- **ランクB**: 二次ソースで確認済み（信頼性は高い）
- **ランクC**: 未確認（要注意）

ランクCのフラグが多い場合、Claudeは追加リサーチを自動実行します。

## AIによる台本レビューとPRレビュー

台本が生成されると、**複数のAIレビュアーが並列でレビュー** します。

**台本品質レビュー**（サブエージェントが担当）:

| レビュアー | チェック内容 |
|-----------|------------|
| ファクトチェッカー（サブエージェント） | 数値・日付・固有名詞の正確性 |
| キャラクターレビュアー（サブエージェント） | もっちー・きなこらしさの一貫性 |
| 構成レビュアー（サブエージェント） | 流れ・わかりやすさ・視聴者への価値 |

指摘があれば自動修正し、問題なければGitHubにPRが作成されます。

**PRレビュー**（PR作成後に自動で実施）:

台本PRのAIレビューは、外部レビュアー（Gemini Code Assist・GitHub Copilot）に依頼する旧フローから、Claude Code完結の多層レビューへ2026年6月に全面移行しました。Gemini Code Assistは2026年7月にサービス自体が終了しています。

| レイヤー | チェック内容 |
|--------|------------|
| L0 | セルフレビュースクリプトとCIによる機械チェック（ハードゲート） |
| L1 | 同一セッションが `/code-review --fix` を実行し、cleared（即マージ可）か negative（要精査）かを判定 |
| L2 | L1がnegativeだった場合、次のセッションがフレッシュな文脈で `/code-review --comment` により精査 |
| L3 | 台本などコンテンツ系PRは、複数の観点を持つエージェントが議論形式で相互レビュー |

:::message
あなたはPRを確認するだけでOKです。
気になる点があればPRコメントで修正を依頼できます。
特に問題なければ自動でマージされ、次のフェーズに進みます。
:::
