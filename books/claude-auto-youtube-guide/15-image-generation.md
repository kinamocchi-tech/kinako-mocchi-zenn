# Phase 4B 画像生成 ── gpt-image-2が一貫したキャラクター画像を作る

## このフェーズでやること

Phase 4Bでは、各シーンに合ったキャラクター画像を **gpt-image-2（OpenAI）** で自動生成します。

実行タイミング: **毎日 12:00 JST**（音声生成完了後）

## なぜ画像生成が必要か

本書のチャンネルは **キャラクターアニメーション形式** の動画を制作しています。

各シーンで:
- もっちーときなこが話している様子
- 技術概念を説明する図解
- 強調したいポイントのビジュアルキュー

などの画像が必要です。これを **gpt-image-2**（OpenAI）が自動生成します。

執筆当初（2026年3〜4月）は、この工程の主エンジンはGemini APIでした。ただし品質と日本語テキスト描画の安定性を比較した結果、2026年7月時点では全用途（キャラクター立ち絵・背景・図解・サムネイル）を **gpt-image-2** に一本化し、Geminiは OpenAI の請求上限到達時のフォールバックに回っています。

## gpt-image-2による画像生成

呼び出し自体は `mcp__gemini-image__generate_image` というMCPツール経由で行います。サーバー名が「gemini-image」のまま残っているのでややこしいのですが、`provider` パラメーターに `openai` を指定することで実体はOpenAIのgpt-image-2が動きます。Claudeはこのツールに対して `model: "gpt-image-2"`・`provider: "openai"`・`quality: "medium"` を渡し、プロンプト文字列で欲しい画像を指示します。

プロンプト例:

```
「ハムスターのもっちーが、驚いた表情でパソコンの画面を見ているイラスト。
ゆるいデフォルメキャラクター風。背景はシンプルな部屋。
同キャラクターの一貫したデザイン（茶色いハムスター、丸い目、メガネなし）」
```

## デザイン定義書による一貫性の維持

複数の動画にわたってキャラクターのビジュアルを一貫させるために、
`docs/design-spec.md`（デザイン定義書）を用意しています。

```markdown
# デザイン定義書

## もっちー（ハムスター）
- 体色: 茶色（#B8834A）
- 目: 黒い丸目
- 体型: ふっくらとした丸いシルエット
- 表情バリエーション: 驚き・喜び・困惑・説明中

## きなこ（セキセイインコ）
- 体色: 黄色+緑のグラデーション
- くちばし: オレンジ
- 翼: 広げた状態と閉じた状態

## 背景設定
- メインシーン: 木の家具のある部屋（ハムスター視点）
- テキスト表示: 画面右側に字幕スタイル
```

Claudeはこの定義書に従って画像プロンプトを生成するため、
毎回異なるキャラクターになることを防げます。

## VisualCueとは

台本JSONには **`visual_cue`（ビジュアルキュー）** というフィールドがあります。

```json
{
  "character": "kinako",
  "text": "Adaptive Thinkingは、タスクの複雑さに応じて...",
  "visual_cue": "スライド：Adaptive Thinkingの仕組みを示す図解"
}
```

このビジュアルキューをもとに、そのセリフに対応した画像が生成されます。

| ビジュアルキューの値 | 意味 |
|-------------------|------|
| `null` | 直前のシーンを継続表示 |
| `""` （空文字列） | 画面をクリア |
| 文字列 | その内容の新しい画像を生成 |

## 画像内テキストの文字化け対策

図解に日本語のテキストを焼き込むと、生成モデルの特性上、確率的に文字が崩れることがあります。これに対しては二段構えで備えています。プロンプト側では焼き込みたい文字列を明示し「一字一句正確に描画すること」という指示を必ず添えます。それでも崩れる可能性はゼロにならないため、生成後にClaudeが画像を確認し、文字化けを検知したら同じプロンプトで再生成する、という運用になっています。

## 生成された画像の確認

生成された画像は以下に保存されます:

```
assets/images/V{動画ID}/{セリフID}_{説明}.png
# 例: assets/images/V001/001_title_card.png
```

GitHubのPRで画像一覧を確認できます。
気になる画像があれば、コメントで修正を依頼できます。
