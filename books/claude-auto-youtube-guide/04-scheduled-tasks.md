# Scheduled Tasksとは ── 毎時自動実行の仕組み

## Scheduled Tasksを一言で言うと

**Scheduled Tasks（スケジュールタスク）** とは、
「毎日〇時にClaudeを自動で起動して、指定した作業をさせる」仕組みです。

スマートフォンの「毎朝7時にアラームを鳴らす」設定と同じように、
「毎日06:00にClaude Codeを起動して、台本を自動生成する」といった設定ができます。

## なぜScheduled Tasksが重要なのか

本書のシステムが「Human-on-the-loop（監視するだけ）」を実現できるのは、
このScheduled Tasksがあるからです。

あなたが何もしなくても、Claudeは毎時決められた作業を自動で実行します（2026年7月時点）。

現行の構成は、単一アカウントが24時間フル稼働する毎時24スロットです（複数アカウントに分けていた運用を2026年6月に1本へ集約しました）。深夜0時から4時台にも、台本・音声・画像・動画の各パイプラインが在庫状況に応じて追加スロットで動きます。代表的なスロットは次のとおりです。

| 時刻（JST） | 自動実行される作業 |
|-----------|------------------|
| 00:00〜04:59 | 深夜帯の追加スロット（audio・image・video・scriptのうち在庫がある工程のみ実行） |
| 06:00 | 台本の自動生成（script-pipeline） |
| 08:00 | ディープリサーチの自律実行（research-runner）→動画ネタの収集（theme-discovery） |
| 09:00・13:00・21:00 | きなこ・もっちーのオーガニックSNS投稿（sns-organic-pipeline） |
| 10:00 | 音声の合成（audio-pipeline） |
| 12:00 | 画像の生成（image-pipeline） |
| 14:00 | 動画の制作（video-pipeline） |
| 19:00 | 公開スケジュール更新・進捗の日次レポート |

さらに、どのスロットも担当タスクに入る前に共通のプリフライトチェックを通します。PRレビューの復帰確認・速報レーンの中断復帰・コメントの即時監視・ベンダー公式リリースの検知など、現在は15種類のリカバリー処理が並んでおり、拾うべき作業があれば担当タスクより先にそちらを片づけます。

## Scheduled Tasksの設定場所

Claude Codeのウェブ画面（claude.ai/code）から設定します。

1. **[Scheduled Tasks]（スケジュールタスク）** メニューを開く
2. **[Add scheduled task]（スケジュール追加）** をクリック
3. **実行間隔** を設定（本書では「1 hour / 毎1時間」）
4. **リポジトリ** を選択（第8章でセットアップしたGitHubリポジトリ）
5. **プロンプト**（指示文）を設定（`docs/rules/hourly-routing.md` に従って動作するよう指示）

:::message
設定後は、あなたが何もしなくてもClaudeが毎時間起動して作業を進めます。週1回Slackのレポートを確認するだけで、チャンネルが継続的に動き続けます。
:::

## 実際のScheduled Tasksの動き方

Scheduled Tasksが起動すると、以下の流れで動作します。

```
1. Claudeが起動 → session-start.sh（フック）が自動実行
   - タイムゾーンをJST（日本時間）に設定
   - VOICEVOXをインストール・起動
   - GitHubへの認証情報を設定

2. CLAUDE.md を読み込む
   - プロジェクトのルール・行動指針を把握

3. 現在時刻（JST）を確認
   - 「06:00」→ 台本生成を実行
   - 「10:00」→ 音声合成を実行
   - それ以外の時刻 → 軽量なタスク（進捗確認など）を実行

4. 作業を実行してGitHubにコミット

5. セッション終了
```

この流れが毎時間繰り返されます。

## セッションとタイムアウト

Scheduled Tasksで起動したClaudeは1つのセッションとして動作します。
長時間の処理（動画制作など）はセッションが途中で終了する場合がありますが、
本書のシステムには **チェックポイント機能** があります。

途中で止まっても、次のセッション開始時に自動で続きから再開します。

## ローカル実行との違い

「自分のPCでプログラムを動かすのと何が違うの？」という疑問があるかもしれません。

| 比較項目 | 自分のPC（ローカル） | Claude Code クラウド |
|---------|------------------|-------------------|
| PC電源が必要 | **必要** | 不要 |
| 24時間自動実行 | できない（PCを常時起動が必要） | **できる** |
| 環境のセットアップ | 自分で行う | **自動**（session-start.sh） |
| 依存関係の管理 | 自分で管理 | **自動** |

本書のシステムはすべてクラウドで動くため、
あなたはPCを開かなくてもコンテンツが自動生成・公開され続けます。

## Scheduled Tasksのコスト

Scheduled Tasksの実行は **Claudeのプランに含まれます**（追加料金なし）。

ただし、毎時実行するとClaudeの利用枠を消費します。
Max プラン推奨の理由の一つがここにあります。
