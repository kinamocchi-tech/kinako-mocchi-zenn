# GitHubとは ── コードとタスクを管理する「司令塔」

## GitHubを一言で言うと

GitHubは **「ファイルのバージョン管理 ＋ タスク管理 ＋ チーム共有」** ができるウェブサービスです。

本書の自動化システムでは、GitHubは **「Claudeへの指示書と作業履歴をすべて保存する場所」** として機能します。

## なぜGitHubが必要なのか

Claudeのクラウド環境は「セッション（接続中の時間）」ごとに動作します。
セッションが終了すると、そのセッション中に作ったファイルは消えてしまいます。

GitHubに保存することで、次のセッションでも前回の続きから作業できます。

また、Claudeは複数のセッションが並行して動くことがあります（毎時の自動実行など）。
GitHubの **「誰が何を変更したか」** の記録が、複数のClaudeが衝突しないための制御にもなっています。

## GitHubの主な機能（本書で使うもの）

### リポジトリ（Repository）

「リポジトリ」はプロジェクトのファイルをまとめて保存する「フォルダ」です。
本書では `kai-kou/kinako-mocchi`（あなたの場合は自分のアカウント名/リポジトリ名）という名前で管理します。

:::message
リポジトリ名は「owner/repo」形式で表記します。`kai-kou` がアカウント名（オーナー）、`kinako-mocchi` がリポジトリ名です。
:::

### Issues（イシュー）

「Issue」はタスクや課題を記録するものです。
本書のシステムでは、**動画のネタ候補・制作進捗・エラーの記録** がすべてIssueで管理されます。

Issueには **ラベル（Label）** を付けられます。
例えば `status:waiting-claude` というラベルは「Claudeが次にやること」を意味します。
Claudeはこのラベルを見て、どのタスクを実行すべきかを判断します。

### ブランチ（Branch）

「ブランチ」はファイルの変更作業を独立した空間で行う仕組みです。
例えば動画V001の台本を作成するとき、`content/V001-ai-agent` というブランチを作り、
そこで作業します。完成したら `main`（本番）ブランチに取り込みます（マージ）。

これにより「制作中の台本」と「完成済みの台本」が混在しなくなります。

### Pull Request（プルリクエスト、PR）

「PR」はブランチの変更を `main` ブランチに取り込む申請です。
本書では、各フェーズ（台本・音声・画像・動画）が完成したときに自動でPRが作成され、
AIがレビューして問題なければ自動でマージされます。

あなたが手動でPRを操作する必要は原則ありません。

### Projects V2（プロジェクト管理）

「Projects V2」はGitHubのKanbanボード機能です。
本書のシステムでは、動画の制作状態を視覚的に管理するために使います。

| ステータス | 意味 |
|----------|------|
| Backlog | ネタ候補（未着手） |
| Todo | 制作予定 |
| In Progress | 制作中 |
| In Review | AIレビュー中 |
| Done | 完了・公開済み |

これらのステータスは **Claudeが自動で更新** します。あなたが手動で動かす必要はありません。

### Milestones（マイルストーン）

「Milestone」は複数のIssueをグループ化して進捗を管理するものです。
例えば「M1: チャンネル開設」「M2: 収益化達成」といったマイルストーンを設定し、
各Issueの完了率を自動で計算してくれます。

### Labels（ラベル）

「Label」はIssueを分類するためのタグです。
本書のシステムでは特に **status:** ラベルが重要です。

| ラベル | 意味 |
|-------|------|
| `status:waiting-user` | **あなたの対応が必要**（週次確認で確認する） |
| `status:waiting-claude` | Claudeが次回起動時に対応する |
| `status:in-progress` | 現在作業中 |

`status:waiting-user` のIssueだけを見れば、
あなたが対応すべきタスクをすぐに把握できます。

## Gitとは

「Git」はGitHubの基盤となる技術で、ファイルの変更履歴を記録する仕組みです。
本書では、GitのコマンドはすべてClaudeが代わりに実行してくれるため、
あなたが直接使う機会はほとんどありません。

:::details Gitコマンドの基本（参考）
最低限知っておくと役立つコマンド:
- `git status`: 変更されたファイルを確認
- `git add ファイル名`: 変更を記録対象に追加
- `git commit -m "メッセージ"`: 変更を記録
- `git push`: GitHubに送信

これらはすべてClaudeが自動で実行します。
:::
