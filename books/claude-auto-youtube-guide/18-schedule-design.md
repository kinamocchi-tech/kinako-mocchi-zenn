# スケジュール設計 ── 06:00〜19:00の自動実行タイムテーブル

## hourly-routing.mdとは

本書のシステムでは、何時に何を実行するかを
`docs/rules/hourly-routing.md` というファイルで定義しています。

Scheduled Tasksが毎時起動するたびに、Claudeはこのファイルを参照し、
「現在の時刻に対応するタスク」を実行します。

## 本書の標準タイムテーブル

| 時刻（JST） | 実行タスク | 内容 |
|------------|----------|------|
| 06:00 | script-pipeline | 台本の自動生成 |
| 07:00（月曜） | 週次マーケティング管理 | バッファ確認・スケジュール更新・週次レポート |
| 07:00（火〜日） | retro-try-handler + project-sync | 改善タスクの実行・Issueの整理 |
| 08:00 | theme-discovery → refinement | ネタ候補の収集・昇格 |
| 09:00 | SNS配信チェック + Shorts確認 | SNS投稿・コンテンツ不足チェック |
| 10:00 | audio-pipeline | 音声の自動合成 |
| 11:00 | project-sync | Issueの整理・PR確認 |
| 12:00 | image-pipeline | 画像の自動生成 |
| 14:00 | video-pipeline | 動画の自動制作 |
| 15:00 | YPP進捗確認 | 収益化条件の進捗チェック |
| 19:00 | 日次進捗報告 | 本日の作業結果をSlackに通知 |

## Claudeが時刻を判断する仕組み

:::message
**重要な技術詳細**

Claudeのクラウドコンテナは世界中のサーバーで動作するため、
タイムゾーンがJST（日本標準時）に設定されていない場合があります。

本書のシステムでは `session-start.sh` で `export TZ=Asia/Tokyo` を実行し、
すべてのコマンドがJST時刻で動作するよう強制しています。

これにより「07:00のタスクが22:00に実行される」といったズレを防いでいます。
（実際にこの問題が発生し、lessons.md L-026 として記録されています）
:::

## タイムテーブルのカスタマイズ

自分のチャンネルに合わせてタイムテーブルを変更できます。

変更方法:
1. `docs/rules/hourly-routing.md` を編集する
2. 変更をGitHubにコミット・プッシュする
3. 次のScheduled Tasks実行から新しいスケジュールで動作する

:::details 一般的なカスタマイズ例
**週2本体制から週3本体制に増やしたい場合**:
- 06:00のscript-pipelineを「未処理テーマが2件以上あれば実行」から「1件以上あれば実行」に変更

**特定の曜日だけ動かしたい場合**:
- hourly-routing.md の各スロットに曜日条件を追加する
  例: `| 06:00 | 月・水・金のみ | script-pipeline |`
:::

## project-syncの役割

`project-sync` は本書のシステムのメンテナンス担当です。

| 処理内容 | 頻度 |
|---------|------|
| 進行中のまま放置されたIssueをリセット | 毎日 07:00・11:00 |
| レビュー待ちのPRを検出・催促 | 全スロット（毎時） |
| 古いブランチをリストアップ | 月曜のみ |

これにより、誰も対応していないタスクが放置されることを防ぎます。
