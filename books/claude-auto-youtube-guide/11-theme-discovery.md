# Phase 1 テーマ発見 ── Claudeが自動でネタを探す

## このフェーズでやること

Phase 1では、Claudeが **毎日自動でテックニュースを収集し、動画のネタ候補をGitHubに登録** します。
あなたが何かをする必要はありません。

実行タイミング: **毎日 08:00 JST**（自動実行）

## theme-discoveryスキルの働き

`theme-discovery` スキルは以下の処理を自動実行します。

1. **ニュースソースの巡回**: HackerNews・海外テックブログ・Arxiv等を自動でチェック
2. **スコアリング**: 視聴者価値・トレンド性・競合状況を自動評価
3. **ネタ候補の登録**: スコアが高いテーマをGitHub Issueとして自動作成

## ネタ候補はどこで確認できるか

生成されたネタ候補は **GitHub Issuesの `phase:1-neta` ラベル** で確認できます。

各Issueには以下の情報が記載されます:

```markdown
## テーマ概要
Claude Sonnet 4.5が新機能「Adaptive Thinking」を搭載

## スコア
- 総合スコア: 18/25
- トレンド性: 5/5（リリース直後）
- 視聴者価値: 4/5（Claude Code利用者に直結）
- 競合状況: 3/5（国内解説が少ない）

## 推奨する動画形式
詳細解説型（15〜20分）

## 関連キーワード
#ClaudeCode #AI #LLM
```

## refinement（昇格）の仕組み

ネタ候補はすべて自動で **スコア評価** されます。
スコアが一定以上（デフォルト: 15点以上）のネタは、
`refinement` スキルによって **Phase 2の制作Issueに自動昇格** します。

```
ネタ候補（phase:1-neta）
  → スコア15点以上: Phase 2の制作Issueへ自動昇格
  → スコア15点未満: バックログとして保留
```

## あなたができること（オプション）

自動選定に任せる場合はそのままで構いません。
ただし、以下のカスタマイズも可能です:

- **手動でネタを追加**: GitHubに手動でIssueを作成し、`phase:1-neta` ラベルを付ける
- **特定テーマを優先**: Issueコメントで「優先してください」と依頼する
- **テーマを却下**: Issueをクローズする（そのネタは処理されなくなる）

:::message
テーマ選定に深くこだわりたい場合は手動介入できますが、
「Claudeに任せてみる」から始めてみることをおすすめします。
:::
