# CLAUDE.mdとは ── Claudeへの「憲法」の書き方

## CLAUDE.mdとは

**CLAUDE.md** は、プロジェクトのルートディレクトリに置くMarkdown形式のファイルです。

Claude Codeは、セッション開始時に自動でこのファイルを読み込み、
「このプロジェクトでどう動くべきか」を把握します。

本書では、CLAUDE.mdを **Claudeへの「憲法」** と呼んでいます。

## なぜCLAUDE.mdが重要なのか

チャットのClaudeに毎回「〇〇してください」と指示するのと同様に、
Claude Codeには毎回同じ指示を与える必要があります。

ただし、自動実行（Scheduled Tasks）の場合は誰も指示を入力できません。
CLAUDE.mdに「プロジェクトのルール」を書いておくことで、
無人でも一貫した動作が実現します。

## CLAUDE.mdに書くべき内容

本書のCLAUDE.mdは主に以下のセクションで構成されます。

### 1. 応答スタイル

```markdown
## 応答スタイル
- **日本語** で応答する
- 過剰な前置きは省略し、本題から入る
```

### 2. プロジェクト概要

```markdown
## プロジェクト概要
YouTubeテック解説チャンネルの制作パイプライン。
```

### 3. プロジェクト大原則（Core Principles）

最重要の行動原則を6つ定義します。これが「憲法」の核心です。

```markdown
## プロジェクト大原則

### CP-1: 自律的判断と行動
ユーザー指示を待たず、自ら判断して行動する。
「やっていいですか？」ではなく「やった結果を報告する」が基本姿勢。

### CP-6: ユーザー介入最小化
定義済みルールの範囲で「〇〇してよいですか？」と質問することは禁止。
実行して結果を報告する。
```

### 4. 判断の優先順位

```markdown
## 判断に迷った時の優先順位
1. 正確性（ファクトチェック未完了なら止める）
2. キャラクターの一貫性
3. 尺（8〜20分）
4. 自動化の効率
```

### 5. 具体的な運用ルール

コミットメッセージの書き方、PRの作成ルール、禁止事項などを記載します。

## CLAUDE.mdの「強制力」の仕組み

CLAUDE.mdに書いたルールは **「お願いベース」** であり、
Claudeが読んで従うことを期待しますが、物理的に強制するわけではありません。

より強い強制力を持たせるために、本書では **4段階のハーネス（制約）システム** を採用します。

| レベル | 種類 | 強制力 |
|-------|------|--------|
| Lv.1 | CLAUDE.md（文書） | 低（Claudeが読む） |
| Lv.2 | セルフレビュースキル（AI検証） | 中 |
| Lv.3 | Hooks（自動実行スクリプト） | **高**（物理的にブロック） |
| Lv.4 | CI/GitHub Actions | **最高**（マージ不可） |

例えば「台本JSONに方言チェック」はLv.3 Hookで実装されており、
ルール違反の台本は物理的に保存できないようになっています。

## CLAUDE.mdを書くときのコツ

1. **具体的に書く**: 「良い台本を書いて」ではなく「1分≒400文字、目標15〜20分」
2. **優先順位を明示する**: 複数の条件が競合したときの判断基準を書く
3. **禁止事項を明記する**: Claudeに「やってはいけないこと」を伝える
4. **徐々に育てる**: 最初は簡素でOK。ミスが起きるたびにルールを追加する

:::message
CLAUDE.mdは一度書いたら完成ではありません。「同じミスが繰り返されたらルールを追加する」という継続的な改善が重要です。本書では `docs/rules/lessons.md`（繰り返しミス防止パターン集）を活用してこのサイクルを自動化しています。
:::

## サブルールファイル（`.claude/rules/`）

CLAUDE.mdだけに全ルールを書くと、ファイルが膨大になります。
本書では、**よく使うルールを別ファイルに分割**して管理しています。

```
.claude/rules/
├── core-principles.md    ← プロジェクト大原則（CP-1〜CP-6）
├── lessons.md            ← 過去のミスから学んだパターン集
├── hourly-routing.md     ← 毎時タスクの実行スケジュール
└── ...
```

これらのファイルはセッション開始時にCLAUDE.mdと一緒に自動で読み込まれます。
CLAUDE.mdに書きたいルールが増えてきたら、別ファイルに切り出すとスッキリします。
