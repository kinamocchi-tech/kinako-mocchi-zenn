# 全体像 ── Claudeのクラウドが一人でチャンネルを回す仕組み

## 「Claudeのクラウドで動く」とはどういうことか

「音声合成も動画制作もClaudeのクラウドで動く」と言われても、
最初はイメージしにくいかもしれません。

まずは、従来の動画制作と本書のアプローチを比較してみましょう。

### 従来の動画制作フロー

```
あなた → テーマ調査 → 台本執筆 → 音声録音 → 動画編集 → アップロード
         （1〜3時間）  （3〜5時間） （1〜2時間） （3〜8時間）  （30分）
         
合計: 1本の動画に 8〜18時間
```

### 本書のアプローチ（自動化後）

```
あなた → Claudeでリサーチ実行
          ↓（約30分）
         Claudeのクラウドが全自動で
         台本生成 → 音声合成 → 画像生成 → 動画制作 → YouTube公開
          ↓
         翌日には動画が公開済み
         
あなたの作業時間: 1本あたり約30分
```

## Claudeのクラウド環境とは

本書で何度も登場する「Claudeのクラウド環境」とは、
Claude Code（後述）が動作するインターネット上のコンピューターのことです。

このコンピューターには以下のような特徴があります。

- **あなたのPCに依存しない**: PCの電源を切っていても動き続ける
- **毎時スケジュール実行**: 設定した時間に自動でタスクが起動する
- **必要なソフトが自動インストールされる**: 起動時に音声合成エンジンや動画制作ツールが自動で準備される

:::message
「音声合成ソフトを自分のPCにインストールしなくていいの？」
はい、不要です。本書の仕組みでは、VOICEVOXという音声合成ソフトがClaudeのクラウドコンピューター上に自動でインストールされ、そこで音声が生成されます。
:::

## 全体パイプラインの流れ

本書で構築するシステムの全体像は以下のとおりです。

```
Phase 1: テーマ発見（Claude 自動）
  Claudeが毎日テックニュースを収集 → 動画ネタ候補をGitHubに登録

Phase 2: ディープリサーチ（あなたが実施）
  Claudeのチャット画面でResearch機能を使い、選んだテーマを深掘り
  → 調査結果をGitHubに保存

Phase 3: 台本生成（Claude 自動）
  リサーチ結果をもとにキャラクター（もっちー・きなこ）の台本を自動生成

Phase 4A: 音声生成（Claude 自動）
  VOICEVOX（音声合成エンジン）で台本を音声に変換

Phase 4B: 画像生成（Claude 自動）
  Gemini APIでシーンに合ったキャラクター画像を生成

Phase 5: 動画制作（Claude 自動）
  Remotion（プログラムで動画を作るツール）で映像を組み立て

Phase 6: 公開・SNS配信（Claude 自動）
  YouTube APIで動画をアップロード、スケジュール公開、SNSに自動投稿
```

## Human-on-the-loop設計

本書のシステムは **「Human-on-the-loop（人間は監視するだけ）」** の設計思想で作られています。

対比として「Human-in-the-loop（人間が各ステップに参加）」があります。

| 設計 | あなたの役割 |
|------|------------|
| Human-in-the-loop | 各ステップで承認・指示する（24時間対応が必要） |
| **Human-on-the-loop** | **システムを監視し、異常時だけ介入する（週15分）** |

本書では後者を目指します。
週に1回、Slackのレポートを確認して問題がなければそのまま。
あとはClaudeが自律的に制作・公開・改善を続けます。

次の章から、このシステムを理解・構築するために必要な基礎知識を学んでいきましょう。
