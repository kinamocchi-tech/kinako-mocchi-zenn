# Phase 5 動画制作 ── Remotionがプログラムで動画を組む

## このフェーズでやること

Phase 5では、生成した音声・画像・字幕データを組み合わせて **完成動画** を自動制作します。

実行タイミング: **毎日 14:00 JST**（画像生成完了後）

## Remotionとは

**Remotion** は「Reactで動画を作る」ツールです。

通常の動画編集（PremiereやFinal Cut Pro）とは全く異なり、
「コード（プログラム）で動画の構成を定義する」アプローチです。

```
通常の動画編集:
  人間がタイムライン上でクリップを並べる → 書き出し

Remotionのアプローチ:
  コードでレイアウト・タイミング・アニメーションを定義 → 自動書き出し
```

この「コードで定義する」特性が、Claude Codeによる完全自動化を可能にしています。

## 動画の構成（レイヤー構造）

Remotionで生成する動画は以下のレイヤーで構成されます。

```
z-index高い（手前）
├── [5] EndingCredits: エンディングテロップ
├── [4] SourceCredit: 参考資料クレジット
├── [3] Subtitle: 字幕（セリフのテキスト）
├── [2] Character: キャラクター画像（もっちー・きなこ）
├── [1] VisualCue: シーン説明・図解画像
└── [0] Background: 背景

z-index低い（奥）
```

各レイヤーは音声タイミングに同期して自動で切り替わります。

## video-pipelineの働き

`video-pipeline` スキルは以下の処理を実行します。

1. **タイムド台本の生成**: 各セリフの開始・終了時間を計算
2. **Remotionデータファイルの生成**: `remotion/src/data/V{ID}/` ディレクトリ（`index.ts`/`config.ts` 等）を自動生成
3. **動画レンダリング**: `npx remotion render` を実行（クラウドで処理）
4. **品質チェック**: 尺・音声同期・無音区間を自動確認
5. **YouTube アップロード**: 完成動画をYouTubeに限定公開でアップロード

## レンダリングにかかる時間

動画のレンダリングはCPUを多く使う処理です。
Claudeのクラウドコンテナで実行されますが、15〜20分程度かかります。

この間、Claudeはバックグラウンドで処理を継続します。

## 字幕の自動生成

字幕は台本JSONのテキストから自動生成されます。

表示ルール:
- 1行あたり最大 **22文字**
- 最大 **3行** まで表示
- 長い文章は自動で分割

:::details 禁則処理とは
日本語の組版ルールで、「、」や「。」が行頭に来ないよう自動調整する処理です。
本書のシステムでは JIS X 4051 に準拠した禁則処理を自動で適用しています。
:::

## 動画品質のチェック

レンダリング・アップロード後、**video-reviewer** スキルが自動でチェックします。

実際のフロー:
1. 動画レンダリング完了
2. YouTube に **限定公開** でアップロード（`tools/upload_video.py` が担当）
3. メタデータ・字幕をアップロード
4. **video-reviewer** が品質を自動チェック
5. 問題なければ公開スケジュールを設定

チェック項目:
- [ ] 動画の尺が8〜20分以内か
- [ ] 音声と字幕が同期しているか
- [ ] 無音区間が長すぎないか（最大2秒まで）
- [ ] エンディングクレジットが正しく表示されているか

問題があれば自動修正し、問題なければ公開スケジュールの設定に進みます。
